# このフォークについて

English: [FORK_NOTES.md](FORK_NOTES.md)

これは Goya Pty Ltd による [BaseElements Plugin](https://github.com/GoyaPtyLtd/BaseElements-Plugin)
のフォークです。プラグイン本体・関数セット・ドキュメントの功績はすべて上流の作者に帰属します。
このフォークは、上流のスコープ外にある2つの具体的なニーズに応えるために存在し、
**API レベルでは可能な限り上流に近い**状態を保ちます。

## なぜこのフォークが存在するか

### 1. 32bit Windows サポート

上流は 2016年10月（v3.3 と v3.3.1 の間）に 32bit Windows（`.fmx`）ビルドのサポートを削除しました。
既存の FileMaker ソリューションの多くは、いまだに古い 32bit の FileMaker 上で動き、
プラグインの関数に依存しています。それらのスクリプトが古い 32bit 専用プラグインを前提にしているため、
64bit への移行がうまくいかないケースがあります。

このフォークは 32bit Windows ビルドを復活させ、プラグインを**移行のブリッジ**として機能させます。
同じプラグインを旧 32bit FileMaker 側と新 64bit FileMaker 側の両方で使えるため、
ソリューションを変換している間も、プラグインに依存したスクリプトを動かし続けられます。

### 2. CJK（日本語・中国語・韓国語）の正しい扱い

プラグインの一部、特にシェル実行やテキスト処理は、CJK 言語環境で正しく動作しないことがあります
（例: システムコマンド出力の文字化け、日本語 Windows でのエンコーディング不整合）。
このフォークは CJK ロケールでの正しい挙動を第一級の目標として扱い、
これらを例外扱いで放置せず修正します。

## 互換性のスコープ（何を保証し、何を保証しないか）

このフォークは上流を **API レベルで追跡し、コードレベルでは追跡しません。**

**同一に保つもの（互換性契約）:**

- **関数セット** — 基にした上流世代と同じ登録関数を公開し、drop-in 置換であり続けます。
  古いソリューションがまだ呼ぶ可能性のある関数を上流が削除している場合、このフォークはそれを残すため、
  既存スクリプトが壊れません。
- **観測可能な挙動** — 各関数は上流と同じ挙動をします。ただし (a) バグ修正 と
  (b) 上記の CJK 対応 は例外で、いずれも意図的な改善です。

**意図的に異なるもの（期待しないでください）:**

- **コードスタイル / 実装** — ソースは大幅にリファクタリングされており、上流と行単位では対応しません。
  内部コードが一致することは期待しないでください。
- **マイナーバージョンの追従** — このフォークは上流の各マイナーリリースを逐一追いません。
  上記2つの目標と安定性に注力し、上流のバージョン履歴を写すことはしません。
